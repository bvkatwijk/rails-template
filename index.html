Variants a.k.a. flavours

  • Base
  • Twitter Bootstrap
  • Sorcery cancancan
  • Ransack Kaminari
  • ActiveJob with Sidekiq

Base 1/2

  • dotenv
  • bower-rails
  • errbit
  • haml
  • rspec stack
  • capistrano
    • bundle-audit
    • database tasks cap production db:pull
    • rails console cap production rails:console

Base 2/2

  • i18n
  • draper
  • foreman (for development)
  • erd rake erd
  • gon (rails variables -> javascript)
  • bullet
  • bin/setup

Config changes

  • default language/locale is dutch/nl
  • environments
    • staging environment inherits from production
    • development uses letter_opener for mail delivery
  • initialisers
    • skip some generators by default
    • log rotate for development and test logs

Scaffold templates

Based on the chosen variants, you get different scaffold templates

Testing/rspec

  • capybara
  • database_cleaner
  • factory_girl
  • shoulda_matchers
  • rake tasks testing
  • time helpers

Some handy bonuses

  • rubocop
  • Insert version and app name in layout as comment

Variant: Twitter bootstrap

  • bootstrap-sass with bootstrap_form
  • extended scaffold templates

Variant: Sorcery cancancan

  • user model and spec's
  • login controller/views
  • extend layout with logged as and logout link
  • extend scaffold templates/controllers to include load_and_authorize_resources and require_login

Variant: Ransack-kaminari

  • if chosen for bootstrap, make kaminari use a bootstrap styled pagination
  • extend scaffold templates to include pagination and sorting

Variant: ActiveJob sidekiq

  • extend forman to boot redis and sidekiq
  • let ActiveJob uses sidekiq as queue adapter
  • mount sidekiq monitoring to /sidekiq using .env username and password
  • add job_helper for rspec
